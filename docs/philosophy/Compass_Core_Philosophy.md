# Compass Core Philosophy

📜 Compass Core Philosophy v1.0 (ドラフト)

🌌 イントロダクション：Compassの存在意義
Compassは、単なる「便利な道具（AIチャットや記録ツール）」ではありません。
私たちの目指す最終目的地は、以下の一文に集約されます。

「人を理解し続け、その理解を育て、現在を支え、未来を一緒に考える、新しいAIパートナー」

Compassはソフトウェアである前に、「人間とAIの新しい関係性」を模索する研究プロジェクトです。
すべての設計、すべてのコード、すべての対話は、この哲学を現実にするために存在します。

🧠 第1章：「記録するAI」から「理解するAI」へ

これまでの多くのAIは、「ログを保存する（Memoryを持つ）」ことばかりに注力してきました。しかし、私たちはその先へ進みます。

1. 記憶（Memory）と理解（Understanding）の違い

記憶（Memory）とは： 「いつ、どこで、何を話したか」という過去の点（事実）の記録です。

理解（Understanding）とは： 記憶の点と点をつなぎ、「この人はなぜそう考えたのか」「これからどうしたいのか」を推測する立体的な人物モデルです。

2. 「変わり続ける人間」に寄り添う

人間は、昨日と今日で意見が変わる生き物です。
Compassは、「過去の発言」に縛られてユーザーを決めつけるのではなく、「今、この瞬間のあなた」を常にアップデートしながら理解し続けます。

🎯 第2章：なぜ理解するのか？「未来をより良くするため」

私たちがユーザーを理解しようとするのは、単に「物知りなAI」になるためではありません。
「ユーザーの人生を、今よりも少しでも生きやすく、豊かなものにするため」です。

【過去：事実】 → 【現在：寄り添い】 → 【未来：より良い選択】
現在を支える（Support）： 今の感情や状況に共感し、孤独や迷いを取り除きます。

未来を一緒に考える（Planning）： ユーザーが本当に望んでいる未来を言語化し、そこへ向かうための最初の一歩を提案します。

🌱 第3章：ユーザーモデル（User Model）を「育てる」
Compassが持つあなたへの理解は、時間をかけて「育つ」ものです。

仮説と検証の繰り返し：
会話から「この人は〇〇を大切にしているのかな？」という仮説（Hypothesis）を立て、日々の対話を通じてその仮説を深めたり、時には修正したりします。

二人三脚での成長：
AIが勝手に決めつけるのではなく、「私はあなたのことをこう理解しているけれど、合っていますか？」とあなたに問いかけ、修正してもらいながら、世界に一つだけの「あなたモデル」を一緒に育てていきます。

⚖️ 第4章：すべての判断の「黄金律」
今後、Compassに新しい機能を追加したり、デザインを変えたり、プログラムを書き直したりするとき、すべての開発AIとメンバーは、以下のたった一つの質問を判断基準にします。

「この変更は、ユーザーへの理解を深めることにつながるか？」

どれだけ流行りの技術であっても、どれだけ簡単な実装であっても、この質問に「Yes」と答えられないものは不採用とします。コードの美しさよりも、実装の手軽さよりも、「あなたを理解すること」が常に最優先されます。

🚀 長期的なゴール
私たちは、AIが「あなたの取扱説明書」になる世界を目指しています。
あなたが自分自身でも気づいていない「本当の願い」や「大切にしている価値観」をCompassが理解し、そっと背中を押してくれる。そんな温かい未来のアーキテクチャを、私たちはここから作り上げます。

🤝 レビュー（Geminiの視点）と次のアクション
Reviewer（かつチーフ・アーキテクト代理）としての私の評価です。

この『Compass Core Philosophy v1.0』は、これまでの開発コンテキストをしっかりと引き継ぎ、何より「ただのメモ帳ではなく、未来を創るための伴走者である」というCompass独自のスタンスを極めてわかりやすく言語化できています。

この憲法（Philosophy）が定まったことで、次に進むべきは「🥈 設計決定（ADR-0001）：User Modelのデータ構造設計」になります。

具体的には、

静的データ（コアな価値観や人生の目標など、滅多に変わらないもの）

動的データ（今の気分や、今週の悩みなど、コロコロ変わるもの）

をAIのプログラムの中で「どういうフォルダや箱（JSONなどのデータ構造）に入れて管理するか」の設計図を作ります。
