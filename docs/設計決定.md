# 設計決定 (Architecture Decision Records)

Compassの設計に関する重要な意思決定を記録します。

## テンプレート

新しいDecisionを追加する際は、以下のフォーマットを一番上（Decisionsのすぐ下）に追加してください。

### D-[番号]: [タイトル]
- **日付**: YYYY-MM-DD
- **ステータス**: Proposed / Accepted / Rejected / Deprecated
- **背景**: なぜこの決定が必要になったのか
- **決定内容**: 何を決定したのか
- **理由**: なぜそのように決定したのか（他の選択肢との比較など）
- **影響範囲**: この決定によって影響を受けるコンポーネントやフロー

## Decisions

### D-0001: ドキュメントの日本語化とDecision管理の導入
- **日付**: 2026-07-15
- **ステータス**: Accepted
- **背景**: Compassプロジェクトの設計・議論は日本語で行われており、「国際化」よりも「開発効率」と「視認性」を優先するため、人間とAIが迷わず理解できるリポジトリを目指す必要があった。
- **決定内容**: `docs/` 配下の主要ドキュメント（ビジョン、設計原則、要件定義、変更履歴、リポジトリガイドなど）のファイル名を日本語化する。設計上の重要な意思決定を記録するため、ADR形式のDecision管理（本ファイル）を導入する。開発フローを「アイデア → 議論 → レビュー → Decision（正式採用） → ドキュメント更新 → 実装 → 変更履歴」へ変更する。
- **理由**: ドキュメントを「人にもAIにも分かりやすい資産」として育て、Founderが迷わず開発を進められるようにするため。将来的なOSS化・英語化は設計が固まった後に検討する。
- **影響範囲**: `docs/` 配下のファイル名、AI Collaboration Protocol、AI Context、リポジトリガイド、変更履歴などの各ドキュメント。

### D-0002: User Model（人物理解）の二層構造と仮説検証システムの設計
- **日付**: 2026-07-15
- **ステータス**: Accepted
- **背景**: Compassのコア思想（Core Philosophy）に基づき、AIは単なる会話ログの保存ではなく「ユーザー理解の継続的なアップデート」を行う必要があります。しかし、人間の特性（長期的な価値観と、短期的な感情や状況の変化）を一つのデータ構造に混在させると、AIの空気の読めない推論（Reasoning）が発生しやすくなります。また、AIが過去の発言からユーザーの性格を決めつける「ハルシネーションの固定化」を防ぐため、すべての理解は「事実（Evidence）に基づく仮説（Hypothesis）」として管理される必要がありました。
- **決定内容**: 
  User Modelを以下の3つの論理レイヤーに分離して設計・実装します。
  
  1. **長期・静的レイヤー（Long-term Layer: "家の柱"）**：価値観（coreValues）、夢や長期目標（longTermGoals）、性格傾向（personalityTraits）などの本質的情報。
  2. **短期・動的レイヤー（Short-term Layer: "吹き抜ける風"）**：現在の気分（currentMood）、直近の悩み（immediateConcerns）、最近の興味（recentInterests）などのリアルタイムな状態。
  3. **仮説・根拠レイヤー（Hypothesis & Evidence Layer: "推理ノート"）**：各データを単なるテキスト値ではなく、確信度（confidence）と、その根拠となった会話ログの参照（evidenceList）を持つ `Hypothesis` オブジェクトとして管理する。

  具体的なデータ構造（TypeScript Schema）は以下の通り定義します。

  ```typescript
  interface UserModel {
    id: string;
    longTerm: {
      coreValues: Hypothesis<string[]>;
      longTermGoals: Hypothesis<string[]>;
      personalityTraits: Hypothesis<string[]>;
    };
    shortTerm: {
      currentMood: {
        status: string;
        intensity: number; // 1-5 (例: 1=軽度、5=極めて重度)
        lastUpdated: string;
      };
      immediateConcerns: Hypothesis<string[]>;
      recentInterests: Hypothesis<string[]>;
    };
  }

  interface Hypothesis<T> {
    value: T;
    confidence: number; // 0.0 - 1.0 (確信度)
    evidenceList: Array<{
      logId: string;
      extractedText: string;
      timestamp: string;
    }>;
    lastUpdated: string;
  }
